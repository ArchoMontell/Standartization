# Documentation Versioning Policy

## Загальні принципи

Документація версіонується разом із проєктом. Кожна стабільна версія документації відповідає певному релізу системи.

## Правила версіонування

- **v1.0** — початкова стабільна версія документації.
- Зміна функціональних вимог у SSD призводить до оновлення мінорної або мажорної версії документації.
- Зміни в архітектурі (SDD, ISD) також потребують оновлення версії документації.
- Незначні редакційні виправлення можуть змінювати патч-версію.

## Актуалізація

Документація оновлюється одночасно зі змінами:
- вимог;
- архітектури;
- API;
- інфраструктури;
- тестових сценаріїв.

## Відповідність релізам

- Версія документації **v1.0** відповідає першому стабільному релізу системи.
- При зміні API або вимог створюється нова версія документації.
