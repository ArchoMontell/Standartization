# SSD - System Specification Document

## 1. Призначення системи
Система забезпечує бронювання квитків через REST API.

## 2. Основні функції

FR-01 — Реєстрація та логін  
FR-02 — Пошук рейсів  
FR-03 — Бронювання  
FR-04 — Оплата  
FR-05 — Скасування  
FR-06 — Історія замовлень  
FR-07 — Адмін керування рейсами  

## 3. Обмеження
- Потрібен інтернет
- Використання REST API
- Залежність від платіжного сервісу

## 4. Нефункціональні вимоги

### Продуктивність
- 500 запитів/хв
- ≤300 мс відповідь

### Доступність
- 99.9%

### Безпека
- HTTPS
- JWT
- Хешування паролів

### Масштабованість
- Горизонтальне масштабування

### Надійність
- Без подвійного бронювання
