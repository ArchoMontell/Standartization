# Single Source of Truth

## Призначення джерел істини

У проєкті визначено такі основні джерела істини для документації:

- **SSD (System Specification Document)** — єдине джерело істини для функціональних і нефункціональних вимог.
- **SDD (Software Design Document)** — єдине джерело істини для програмної архітектури, структури сервісів, API та моделі даних.
- **ISD (Infrastructure Specification Document)** — єдине джерело істини для інфраструктурної архітектури, середовищ розгортання та технічних платформ.
- **Test Strategy** — єдине джерело істини для підходів, рівнів і принципів тестування.
- **Traceability Matrix** — єдине джерело істини для відповідності між вимогами та тестовими сценаріями.

## Правила оновлення документації

1. Усі функціональні вимоги спочатку оновлюються в SSD.
2. Якщо зміна вимог впливає на архітектуру, відповідні зміни вносяться в SDD та/або ISD.
3. Після зміни вимог обов’язково оновлюється Traceability Matrix.
4. Якщо змінюється підхід до тестування, оновлюється Test Strategy.
5. Документація змінюється через pull request разом із відповідними змінами в коді.

## Правило узгодженості

Жоден документ не повинен дублювати критичну інформацію без посилання на основне джерело істини. У разі конфлікту пріоритет має документ, визначений як SSOT.
