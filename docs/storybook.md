# Візуальна документація frontend (Storybook)

Цей документ описує приклади візуальної UI-документації у форматі Storybook для системи бронювання квитків.

## Компонент `FlightCard`
Цей компонент відповідає за відображення інформації про рейс (місце призначення, час, ціна) у вигляді картки в списку результатів пошуку.

### Стан: Default
- **Опис:** Стандартне відображення доступного рейсу. 
- **Візуалізація:** Картка містить чіткий логотип авіакомпанії, час вильоту та прильоту, ціну та активну кнопку "Обрати".

### Стан: Sold Out (Disabled)
- **Опис:** Відображення рейсу, на який всі квитки розпродано.
- **Візуалізація:** Картка має сірий (неактивний) фон. Кнопка "Обрати" змінена на "Немає місць" і деактивована.

### Стан: Loading
- **Опис:** Скелетон картки, що відображається під час завантаження даних про рейси з API.
- **Візуалізація:** Сірі прямокутники із легкою пульсуючою анімацією на місці тексту та кнопок.

## Компонент `BookingForm`
Форма для введення даних пасажира та оформлення бронювання.

### Стан: Default
- **Опис:** Форма з порожніми полями (Ім'я, Прізвище, Документ) готовими до введення.
- **Візуалізація:** Поля введення з плейсхолдерами, кнопка "Підтвердити бронювання" неактивна, якщо не заповнені обов'язкові поля.

### Стан: Error (Validation Failed)
- **Опис:** Стан форми, коли користувач ввів некоректні дані (наприклад, ім'я містить цифри або пропущено серію паспорта).
- **Візуалізація:** Проблемні поля виділені червоною рамкою, під ними з'являються повідомлення з поясненням помилки. Кнопка підтвердження деактивована.

### Стан: Submitting (Loading)
- **Опис:** Стан відправки запиту `POST /bookings` на сервер.
- **Візуалізація:** На кнопці "Підтвердити" відображається спінер завантаження. Всі поля форми стають `disabled`, щоб запобігти дублюванню запитів.
