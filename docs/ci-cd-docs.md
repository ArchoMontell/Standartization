# Автоматизація документації в CI/CD

Даний документ регламентує процес автоматизації збірки, валідації та публікації всіх видів документації проекту (API, UI, загальна документація).

## Артефакти документації
Наступні артефакти підлягають автоматичній генерації під час CI/CD процесу:
1. **API-документація**: сторінки Swagger UI на базі `openapi.yaml`.
2. **UI-документація**: статика Storybook для frontend-додатку.
3. **Загальний сайт документації**: сайт на базі MkDocs.

## Тригери генерації
- **Push у `main` гілку**: запускає повний цикл (збірка + публікація) для оновлення production-документації.
- **Push у `feature/*` гілки**: запускає етап валідації і збірки (без публікації) для перевірки цілісності (Linting, Schema Validation) в межах Pull Request-у.

## Пайплайн (Pipeline Steps)

### 1. Валідація (Lint & Schema Validation)
- Для OpenAPI використовується Spectral: `spectral lint docs/openapi.yaml`. Гарантує, що контракт відповідає корпоративним та галузевим стандартам REST.
- Для Markdown файлів (MkDocs) використовується Markdownlint.

### 2. Збірка документації (Build)
- **MkDocs Build**: `mkdocs build` генерує статику для загального сайту.
- **Swagger UI**: завантаження та інтеграція `swagger.html` та статики у папку `site` поруч з MkDocs.
- **Storybook Build**: `npm run build-storybook` у фронтенд-модулі генерує компоненти.

### 3. Публікація (Publish)
- Налаштовано деплой (публікацію) скомпільованої папки `site` та `storybook-static` за допомогою **GitHub Pages**.
- Структура URL:
  - `https://org.github.io/project/docs/` — загальна документація;
  - `https://org.github.io/project/api/` — Swagger UI;
  - `https://org.github.io/project/storybook/` — Storybook UI компоненти.

## Політика Versioning документації
- Версія загальної документації та API (через версію в полі `info.version` OpenAPI-файлу) завжди відповідає поточній релізній версії продукту (наприклад, 1.2.0). 
- **Breaking changes**: Будь-яка зворотньо-несумісна зміна API викликає створення нової major-версії API (v2) і, відповідно, генерацію паралельного масиву документації `https://org.github.io/project/api/v2/`.
